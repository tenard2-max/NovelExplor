# NovelExplor 요건정리서

> 목적: GitHub Pages 기반 NovelExplor를 다른 개발자가 **구조·권한·저장소·동기화**를 이해하고 이어서 개발할 수 있도록 정리한 문서입니다.  
> 기준: 저장소 `main` 브랜치 코드 (`index.html` + `js/` + `css/` + `data/workspace/`)  
> 성격: 서버리스 정적 웹앱 (백엔드 없음). 인증·ACL·GitHub 동기화는 **브라우저 + GitHub REST API**로 동작합니다.

---

## 1. 한 줄 요약

| 항목 | 내용 |
|------|------|
| 런타임 | 브라우저 (Chrome/Edge 권장), GitHub Pages 배포 |
| 로컬 상태 | IndexedDB (`FantasyForeshadowDB`) + localStorage |
| 원격 상태 | GitHub 저장소 `data/workspace/` (JSON 스냅샷 + PNG/MD 오버레이 + XML) |
| 권한 | 로그인 역할 + 프로젝트별 `writers` / `writerUsernames` |
| GitHub 쓰기 | PAT(`repo`) — **마스터만 UI에서 등록**, origin(호스트)마다 별도 보관 |
| 핵심 원칙 | **열기 ≠ 자동 푸시**. GitHub 반영은 「프로젝트 저장」/기본 저장/수동 Sync |

## 2. 시스템 구성도

```text
┌─────────────────────────────────────────────────────────────┐
│  Browser (origin 단위: github.io / localhost / 127.0.0.1)   │
│  ┌──────────────┐  ┌──────────────┐  ┌───────────────────┐ │
│  │ localStorage │  │ IndexedDB    │  │ Sync Folder API   │ │
│  │ PAT, 세션,   │  │ 프로젝트·인물│  │ (선택) JSON 파일  │ │
│  │ TTS, 테마…   │  │ 스토리·설정  │  │ 디렉터리 핸들     │ │
│  └──────────────┘  └──────────────┘  └───────────────────┘ │
│            │ save / pull / hydrate                              │
│            ▼                                                    │
│     GitHub REST API  (Bearer PAT, rate limit 추적)             │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
              github.com/{owner}/{repo}/data/workspace/
```

**Origin 주의:** `localhost`와 `127.0.0.1`, GitHub Pages는 IndexedDB·PAT가 **서로 공유되지 않습니다.**

## 3. 역할(관리자·사용자) 개념

구현: `js/core/auth.js`, `js/core/auth-sync.js`, `js/ui/permissions.js`

### 3.1 역할 코드

| role | 표시명 | 요약 |
|------|--------|------|
| `user` | 사용자 | 열람 중심. 생성·저장·업로드 불가 |
| `admin_novelist` | 소설가 | `writers`에 포함된 프로젝트만 편집 |
| `admin_developer` | 개발자 | 소설가와 동일 ACL, 개발 역할 구분 |
| `admin_master` | 마스터관리자 | 전 프로젝트 쓰기, 사용자·GitHub·프로젝트 관리 |

기본 마스터: username `master` (최초 로컬 fallback 비밀번호 `master`, `mustChangePassword: true`).

### 3.2 사용자 저장소 (GitHub SoT)

경로: `data/workspace/auth/users.json`  
포맷: `novel-explor-users/v1`

| 필드 | 설명 |
|------|------|
| `id` | 사용자 ID |
| `usernameHash` | pepper 포함 SHA-256 (조회용) |
| `usernameEnc` | AES-GCM 암호화 username |
| `passwordHash` / `salt` | PBKDF2-SHA256, 120,000 iterations |
| `role` | 위 역할 중 하나 |
| `mustChangePassword` | 강제 변경 여부 |
| `disabled` | 비활성 |
| `createdAt` / `updatedAt` | 시각 |

레거시: `data/workspace/auth/master-auth.json` (`master-auth/v1`).

> 인증은 **서버 세션이 아니라** 공개 JSON + 클라이언트 검증입니다. pepper는 앱 상수이므로 “서버급 비밀관리”는 아닙니다.

### 3.3 세션

- localStorage 키: `ne-auth-session`
- 값: `{ userId, role, loginAt }` (비밀번호·PAT 미포함)
- 복원 시 IndexedDB `users`에서 재조회. `disabled`면 거부.

### 3.4 ACL (프로젝트 쓰기)

핵심: `canManageProjectContent(project, user)`

| 조건 | 결과 |
|------|------|
| 미로그인 / `user` | 쓰기 불가 |
| `admin_master` | 항상 쓰기 가능 |
| 소설가·개발자 + `project.writers[]`에 `user.id` | 쓰기 가능 |
| 소설가·개발자 + `writerUsernames[]`에 username(소문자) | 쓰기 가능 |
| writers / writerUsernames가 있는데 불일치 | 거부 |
| ACL 목록이 모두 비어 있고 `ownerId === user.id` | fallback 허용 |

**읽기:** 로그인 사용자는 로컬 IDB·JSON·GitHub 프로젝트를 열 수 있습니다.  
쓰기 불가 시 UI에 `(열람만가능)`, `project-readonly` 클래스, 편집 도구 숨김.

**마스터 전용 UI 예:**

- GitHub 패널(PAT 저장·테스트·수동 Sync/Pull)
- 프로젝트 관리(쓰기 권한 부여, 스냅샷 목록/삭제, default/latest, 고아 자산 정리, 제목 이력)

### 3.5 역할별 화면 게이트 (요약)

| 기능 | 사용자 | 소설가/개발자 | 마스터 |
|------|--------|---------------|--------|
| 프로젝트 열기·열람 | ○ | ○ | ○ |
| 콘텐츠 수정 | × | writers인 프로젝트만 | ○ |
| 프로젝트 저장 → GitHub | × | writers + 해당 origin에 PAT 있으면 가능* | ○ |
| PAT 등록 UI | × | × | ○ |
| 역할·프로젝트 관리 | × | × | ○ |

\* PAT UI는 마스터만. 같은 origin에 이미 PAT가 있으면 저장 경로의 GitHub 업로드는 코드상 가능할 수 있음.

## 4. 로컬 브라우저 저장

### 4.1 IndexedDB — `FantasyForeshadowDB` (v4)

구현: `js/core/storage.js`  
기본 키: `{ keyPath: 'id' }`

| Store | 용도 | 인덱스 |
|-------|------|--------|
| `projects` | 프로젝트 메타 | `ownerId` |
| `stories` | 소설 읽기 ST 본문 | `projectId` |
| `episodes` | 스토리 네비 EP | `projectId` |
| `characters` | 인물 + Data URL 이미지 | `projectId` |
| `sceneCuts` | 장면컷 + 이미지 | `projectId` |
| `worlds` | 세계관 | `projectId` |
| `foreshadows` | 복선 | `projectId` |
| `timeline` | 타임라인 | `projectId` |
| `files` | 가상 워크스페이스 파일 | `projectId` |
| `settings` | 관계도·배치·월페이퍼·이력 등 | — |
| `users` | 인증 캐시 | `username` unique |
| `versions`, `trash` | 스키마만 존재(실질 미사용에 가깝음) | — |

주요 `settings.id` 예:

- `${projectId}-character-relations`
- `${projectId}-character-layout`
- `${projectId}-canvas-wallpaper`
- `app-default-project` / `app-default-project-backup`
- `project-title-history`
- `orphan-cleanup-history`

**이미지:** 로컬에서는 `avatarDataUrl`, `images[]`, 장면컷 `image`, 월페이퍼 `dataUrl` 등 **Data URL**.  
GitHub split 스냅샷에서는 Data URL을 제거하고 `avatarPath` / `imagePaths` / `imagePath` / `dataPath`로 치환.

### 4.2 동기화 폴더 핸들 DB

- DB: `NovelExplorSyncFolder`
- Store: `handles`, key `project-sync-dir`
- 값: `FileSystemDirectoryHandle` (`js/core/sync-folder.js`)
- Chrome/Edge File System Access API. 폴더 **루트에 JSON 파일만** 기록(하위 폴더 생성 안 함).

### 4.3 localStorage 키

| 키 | 용도 |
|----|------|
| `fft-github-config` | `{ owner, repo, branch, workspaceRoot, token }` PAT 포함 |
| `ne-auth-session` | 로그인 세션 |
| `ne-last-opened-project-id` | 마지막 연 프로젝트 |
| `fft-backup-${projectId}` | 자동 백업 JSON |
| `fft-backup-meta` | 백업 메타 |
| `fft-theme` | `dark` / `light` |
| `ne-save-theme` | 저장 파일명 테마 |
| `ne-save-update-title` | 저장 시 제목 갱신 `'1'/'0'` |
| `ne-save-to-folder` | 폴더에도 저장 `'1'/'0'` |
| `fft-json-version` | 마지막 snapshot 스탬프 |
| `fft-upload-version` | 마지막 overlay 업로드 스탬프 |
| `ne-tts-voice-uri` | TTS 음성 |

**Ctrl+Shift+R 로는 PAT·세션이 지워지지 않습니다.** (사이트 데이터 삭제 / 다른 origin / 시크릿 종료 시 소실)

## 5. GitHub 저장소 구조 (`data/workspace/`)

```text
data/workspace/
├─ workspace.xml
├─ auth/
│  ├─ users.json
│  └─ master-auth.json          # 레거시
├─ snapshots/
│  ├─ YYYYMMDDHHMMSS[_테마].json
│  ├─ latest.json                 # 포인터
│  ├─ default.json                # 기본 프로젝트 포인터
│  └─ README.md
├─ overlays/
│  ├─ upload-latest.json
│  ├─ wallpaper.png               # (있을 때)
│  ├─ characters/
│  │  ├─ CHR####.png              # 대표 아바타
│  │  ├─ CHR####_N.png            # 갤러리
│  │  └─ CHR####_이름.md
│  ├─ scene-cuts/
│  │  └─ SCN####.(png|jpg|webp)
│  ├─ stories/                    # ST/EP 등 .md/.txt
│  └─ files/
└─ sections/
   ├─ 03_characters.xml
   ├─ 04_foreshadows.xml
   ├─ 05_timeline.xml
   ├─ 10_reader.xml
   ├─ 11_story_nav.xml
   └─ (workspace.xml은 루트에도)
```

기본 config (`js/core/github-config.js`):

- owner / repo / branch / `workspaceRoot: data/workspace`
- Pages URL이면 owner·repo를 URL에서 추론 가능

### 5.1 스냅샷 규칙

- 파일명: `^(\d{14})(?:_([^.]+))?\.json$`
- 본문 format: `novel-explor-github-snapshot`, `version: 1`, `backupType: 'split'`
- `latest.json`: `{ snapshotId, filename, updatedAt, reason }`
- `default.json`: 위 + `title`, `reason: 'default'`

### 5.2 동기화 시 갱신되는 XML

`buildSectionXmlFiles()`가 매 동기화 때 다루는 것(동적 section 5 + workspace manifest):

- `03_characters.xml`, `04_foreshadows.xml`, `05_timeline.xml`
- `10_reader.xml`, `11_story_nav.xml`
- `workspace.xml`

`workspace.xml`에는 더 많은 섹션 ID가 등록될 수 있음 (`00_master` … `12_editor` 등).

## 6. 저장·동기화 흐름 (MD + PNG + JSON)

### 6.1 「프로젝트 저장」이 하는 일

구현 축: `js/core/backup.js`, `js/core/github-sync.js`, `js/core/sync-folder.js`

1. 테마/옵션 다이얼로그 (제목 갱신, 폴더 저장 ON/OFF 등)
2. `buildBackupPayload()` → 로컬 full(또는 lite) 백업
3. **폴더**: File System Access로 `YYYYMMDDHHMMSS[_테마].json` 기록 (실패 시 다운로드)
4. **localStorage** 자동 백업 (`fft-backup-*`, 용량 초과 시 `backupType: 'lite'`)
5. **PAT 있으면** `syncProjectToGithub({ reason: 'save' })`
   - Data URL → PNG blob 분리 → `overlays/characters/` 등
   - split 스냅샷 JSON + `latest.json` (+ 옵션 시 `default.json`)
   - section XML
   - `commitRepoFiles`: blob(배치 4) → tree → commit → ref PATCH  
     non-fast-forward 시 tip 재조회 후 재시도
6. API 잔량 `< 500`이면 저장 전 경고 확인창 (`GITHUB_RATE_LIMIT_WARN_THRESHOLD`)

### 6.2 GitHub에 올리는 파일 수 ≈ API 비용

대략: **고정 ~5회(ref/commit/tree/patch) + 파일(blob) 수**.  
인물 PNG·스토리 MD가 많으면 수십~수백 호출/저장.

### 6.3 자동 GitHub 푸시를 하지 않는 경우 (중요)

| 동작 | GitHub 자동 푸시 |
|------|------------------|
| 로컬 IndexedDB 프로젝트 열기 | 없음 |
| 로컬/폴더 JSON 열기 | 없음 (`skipGithub: true`, `apply-json` 자동 sync 제거됨) |
| 편집·이미지 변경만 | 없음 (`scheduleGithubSync`는 **의도적 no-op**) |
| 「프로젝트 저장」/기본 저장/패널 수동 Sync | **있음** (PAT 필요) |

> 과거에는 로컬 JSON 열기만으로 전체 푸시가 2회+이벤트 폭주가 나와 API를 대량 소모하는 버그가 있었습니다. 현재는 차단된 상태입니다.

### 6.4 GitHub에서 열기 / Hydration

1. `pullProjectFromGithub()` → `latest.json` 또는 지정 스냅샷
2. split JSON을 IDB에 복원 (`avatarPath` 등 경로 유지, 즉시 Data URL 대량 로드 아님)
3. 화면에서 미디어가 필요할 때 `resolveMediaSrc` / `background-assets`가 raw PNG를 fetch·캐시  
   - Cache: `ne-project-assets-v1`, 동시 다운로드 제한 있음
4. `github-hydrate`: `avatarPath` 없더라도 `overlays/characters/{characterId}.png` 관례 경로 fallback

### 6.5 크로스 브라우저 이미지

- Edge에서 이미지 등록 후 **저장(동기화)** 해야 Chrome 등 다른 브라우저에서 GitHub 경로로 보임.
- 예전 `uploadOnly`만으로는 스냅샷의 `avatarPath`가 안 맞을 수 있었고, 복원 fallback·스냅샷 갱신으로 보완됨.
- 다른 PC·휴대폰: **PAT를 그 브라우저 origin에 다시 등록**해야 GitHub 쓰기가 됨. pull로 PAT가 전파되지 않음.

## 7. 데이터 모델 (백업 JSON)

### 7.1 로컬 full 백업 (`buildBackupPayload`)

```json
{
  "format": "foreshadow-backup",
  "version": 1,
  "backupType": "full | lite",
  "project": { },
  "stories": [],
  "episodes": [],
  "characters": [],
  "sceneCuts": [],
  "worlds": [],
  "foreshadows": [],
  "timeline": [],
  "files": [],
  "characterRelations": {},
  "settings": {},
  "exportedAt": "ISO-8601"
}
```

GitHub split: `format: "novel-explor-github-snapshot"`, `backupType: "split"`, `snapshotId` 추가.

### 7.2 Project

`id`/`projectId`, `title`, `author`, `createdAt`/`updatedAt`, `ownerId`,  
`writers[]`, `writerUsernames[]`, `workspace`, `language`, `versionMeta` 등.

### 7.3 Character

- 로컬: `avatarDataUrl`, `images[]`
- 원격 참조: `avatarPath`, `imagePaths[]`
- `characterId`: `CHR####`, `name`, `alias[]`, 프로필 필드들

### 7.4 Story / Episode / File

| 구분 | 용도 | 파일 관례 |
|------|------|-----------|
| Story | Reader 본문 | `ST###.md`, `stories.content` |
| Episode | Story Nav·분석 | `EP###.md` |
| File | 가상 워크스페이스 | `path`, `content` / GitHub에선 `contentPath` |

## 8. PAT · Rate Limit

| 항목 | 내용 |
|------|------|
| 저장 | `fft-github-config.token` (평문, origin 스코프) |
| UI 등록 | 마스터만 (`assertMasterGithub`) |
| 권한 | Classic: `repo` / Fine-grained: Contents R/W |
| 한도 | 인증 REST ≈ 5000/시간, 비인증 ≈ 60/시간 |
| UI | Token 옆 `remaining/limit`, Branch 옆 리셋 남은 `분:초` |
| 경고 | 저장 전 remaining &lt; 500 → 확인 다이얼로그 |
| Metrics | `js/core/github-metrics.js` — API/RAW 호출, 경고 |

## 9. 부트 시퀀스

구현: `js/main.js` → `boot()`

1. PWA·metrics·인증 gate 초기화, `auth-locked`
2. UI 모듈 초기화, sync-folder 핸들 복원
3. `workspace.xml` 로드 시도
4. `initAuth()` — 세션 복원, 사용자 catalog pull/merge, 마스터 보장
5. 미로그인 → 로그인 게이트
6. 마스터면 orphan 프로젝트 claim
7. `listProjects()` → `resolveBootProjectId()`  
   (`ne-last-opened-project-id` 우선, 없으면 `updatedAt` 최신)
8. 있으면 `loadProject()`
9. 없으면: 일반 사용자 → default 프로젝트 시도 / 관리자 → 로컬 복원 제안 또는 시드 「인류 생존 프로젝트」
10. origin 경고, 버전 포인터 갱신, 역할별 초기 화면

페이지 이탈 시 dirty면 autosave flush (`visibilitychange` / `pagehide` / `beforeunload`).

## 11. 개발자가 지켜야 할 규칙

1. **열기만으로 GitHub 전체 푸시하지 말 것.** 명시적 저장/Sync만.
2. **PAT·IndexedDB는 origin 단위.** 문서/안내에 `localhost` vs `127.0.0.1` vs Pages를 구분할 것.
3. **이미지 크로스기기:** Data URL(로컬) ↔ path(원격) 변환과 hydrate/fallback을 깨뜨리지 말 것.
4. **ACL:** import/열기로 `ownerId`/`writers`를 가로채지 말 것.
5. **Rate limit:** 대량 blob 업로드·루프 sync·불필요한 `listRepoDir` 반복 금지. 잔량 경고 유지.
6. **인증 보안 한계:** 클라이언트 검증임을 전제로, 민감 운영은 PAT 노출·users.json 공개 범위를 인지할 것.
7. **기술 스택 기본:** 이 앱은 vanilla JS + GitHub Pages. 프론트 프레임워크 전환은 별도 합의.
